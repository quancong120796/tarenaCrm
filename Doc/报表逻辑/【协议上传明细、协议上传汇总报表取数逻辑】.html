【协议上传明细、协议上传汇总报表取数逻辑】

    • 协议上传进度明细 （月报日期在查询条件范围内）
      • 学历证明 1-已上传 0-未上传 -1 无需上传
        • 以下情况为从学员资料表中关联学员编码查询学历证明(stu_attachment表type=802)
          • 系列班级包含  UEDTN、JSDTN、NSDTN  && 合同的课期 >= 4 &&(学员身份证为大陆身份证且年龄小于等于21或大于等于30  或身份证类型不为大陆身份证 )
          • 系列班级包含  WEBTN、CGBTN、AIDTN、BIGTN、CSDTN、ESDTN  && 合同的课期 >= 4 
          • 系列班级包含  UCDTN 合同的课期 >= 3   &&(学员身份证为大陆身份证且年龄小于等于21或大于等于30  或身份证类型不为大陆身份证 )
        • 否则取值为-1 即无需上传
      • 中心合同签字页
        • 学员的中心合同为线上(中心合同的contract_id 不为空)
          • 合同表的契约锁签署状态(qys_state='COMPLETE') 为已完成 则为已上传，否则为未上传
        • 学员的中心合同为线下(中心合同的contract_id 为空)
          • 关联学员编码查询学员资料表是否存在中心合同签字页(type=803),若存在则为已上传，否则为未上传
      • 软件合同签字页
        • 学员的软件合同为线上(软件合同的contract_id 不为空)
          • 合同表的契约锁签署状态(qys_state='COMPLETE') 为已完成 则为已上传，否则为未上传
        • 学员的软件合同为线下(软件合同的contract_id 为空)
          • 关联学员编码查询学员资料表是否存在软件合同签字页(type=804),若存在则为已上传，否则为未上传
      • 无需就业证明
        • 关联学员编码查询学员资料表是否存在无需就业证明(type =805) ,若存在则为已上传，不存在则为未上传
      • 报名确认书
        • 中心合同为线上时，若存在 电子合同创建时间>= 2019-11--06   则为已上传 
        • 中心合同不为线上时，关联学员编码查询学员资料表是否存在报名确认书(type =806),若存在则为已上传
        • 在以上两个条件基础上若系列班级为'JN'，'GN' 则无需上传
      • 身份证
        • 关联学员编码查询是否存在身份证(type =801),若存在则为已上传
      • VIP进班确认单
        • 关联学员编码查询学员资料表是否存在VIP进班确认单(type =808),若存在则为已上传
    • 协议上传进度汇总 
      • 中心合同签字页 上传数
        • 明细中 中心合同签字页 为已上传的和
      • 软件合同签字页 上传数
        • 明细中 软件合同签字页 为已上传且软件合同不为空 的和
      • 报名确认书 上传数
        • 明细中明细中 报名确认书为已上传的和 
      • 身份证
        • 明细中身份证上传数之和
      • VIP进班确认单
        • 明细中VIP进班确认单上传数之和
      • 无需就业证明 上传数 
        • 明细中无需上传为已上传且 学员的就业状态为空或者不为推荐就业  的数量和
      • 学历证明 
        • 明细中学历证明为已上传的和 
      • 协议已完成人数
        • 中间数据 完成数
          •  身份证：取值明细中的身份证（上传为1,未上传为0,下同）
          • 中心合同签字页：取值明细中的中心合同签字数
          • 软件合同签字页:不存在软件合同或软件合同学费值为0时 取值1，否则取值明细中的软件合同签字页上传数
          • VIP进班确认单：若学员合同系列班不包含'VN','OT','VT',取值1，否则取值明细中的VIP进班确认单
          • 报名确认书:若学员系列班级包含'JN','GN',取值1 否则取值明细中的VIP进班确认单
        • 取值规则
          • 若中间数完成数之和为5  则该学员已完成所有资料上传，否则为未完成
          • 即对应学员 身份证+中心合同签字页+软件合同签字页+VIP进班确认单+报名确认书 = 5 为完成
